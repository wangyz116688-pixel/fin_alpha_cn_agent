# 因子与选股策略说明

本文档说明 AlphaAgent 当前最新量化层逻辑。旧版中“XGBoost 为默认打分模型”“最多持仓 5 只”“`C_mixed = xgb_score`”等描述已经过时。当前最终默认策略为：

```text
score_method = train_ic_blend
allocation_method = score
position_count = 3
max_drawdown_stop = 0
```

## 1. 策略设计原则

AlphaAgent 的量化层遵循三条原则：

1. 可复现：因子和回测流程均为确定性计算。
2. 防未来函数：T 日交易只使用 T-1 日及以前的数据。
3. 可解释：最终排序来自经过 IC 验证的因子方向，而不是单纯黑箱模型输出。

## 2. 数据基础

| 项目 | 说明 |
|---|---|
| 数据频率 | 日频 |
| 价格口径 | 前复权 K 线 |
| 主数据源 | baostock |
| 备用数据源 | AKShare |
| 回看窗口 | 因子计算使用约 90 日窗口，并保留额外缓冲 |
| 回测模式 | 支持离线缓存复现 |

离线缓存加载已支持非交易日起止日期。例如 `2024-01-01` 不是交易日，但系统会正确使用 2024 年第一个真实交易日开始的缓存数据。

## 3. 因子体系

当前系统主要使用日频量价因子与 AlphaNet 风格时序特征。

### 3.1 动量与反转

| 因子 | 含义 |
|---|---|
| `mom_1d` | 1 日动量 |
| `mom_5d` | 5 日动量 |
| `mom_20d` | 20 日动量 |
| `rev_1d` | 1 日反转 |
| `ret_close_10` | 10 日收盘价收益 |

### 3.2 波动类因子

| 因子 | 含义 |
|---|---|
| `vol_5d` | 5 日收益波动 |
| `vol_20d` | 20 日收益波动 |
| `vol_ratio` | 短期波动相对中期波动 |
| `std_close_10` | 10 日价格标准差 |
| `std_vol_10` | 10 日成交量标准差 |

### 3.3 成交量与换手类因子

| 因子 | 含义 |
|---|---|
| `turn_1d` | 单日换手率 |
| `turn_5d_avg` | 5 日平均换手率 |
| `turn_ratio` | 单日换手相对 5 日均值 |
| `zscore_vol_5` | 成交量 5 日标准化偏离 |
| `decay_vol_5` | 成交量 5 日线性衰减均值 |

### 3.4 价量关系因子

| 因子 | 含义 |
|---|---|
| `price_vol_corr_5` / `corr_close_vol_5` | 5 日价格与成交量相关性 |
| `price_turn_corr_5` / `corr_close_turn_5` | 5 日价格与换手率相关性 |
| `corr_return_vol_5` | 收益与成交量相关性 |
| `vwap_dev` | 收盘价相对 VWAP 偏离 |
| `decay_close_5` | 价格 5 日线性衰减均值 |

## 4. 因子有效性验证

系统使用 IC 分析判断因子方向。IC 是因子值与下一交易日收益之间的秩相关系数。IC 越稳定，说明该因子越可能提供可复现的排序信息。

相关脚本：

```powershell
.\venv\Scripts\python.exe scripts\analyze_factor_ic.py `
  --start 20260515 `
  --end 20260713 `
  --offline `
  --cache-dir data\prefetch_cache `
  --factor-cache-dir data\factor_cache `
  --max-stocks 500
```

最终项目没有继续使用单一 XGBoost 分数作为默认策略，而是采用训练期 IC 方向组合成 `train_ic_blend`，原因是它在多窗口回测中更稳健、更容易解释。

## 5. 当前默认排序方法：train_ic_blend

`train_ic_blend` 使用训练期 IC 验证后的因子方向。方向和权重来自训练集统计结果，核心思想是：

- 正 IC 因子：因子越高，预期下一日相对表现越好。
- 负 IC 因子：因子越低，预期下一日相对表现越好。

当前组合中使用的主要方向包括：

| 因子 | 方向 |
|---|---|
| `turn_5d_avg` | 负向 |
| `mom_20d` | 负向 |
| `vol_20d` | 负向 |
| `vwap_dev` | 负向 |
| `zscore_vol_5` | 负向 |
| `turn_ratio` | 负向 |
| `rev_1d` | 正向 |
| `std_vol_10` | 正向 |
| `std_close_10` | 正向 |
| `decay_vol_5` | 正向 |

这说明当前策略更偏向捕捉短期反转、适度活跃和风险可控的标的，而不是盲目追逐高动量、高换手或高波动股票。

## 6. 选股流程

每日回测或选股流程：

1. 确定股票池。
2. 读取或拉取历史 K 线。
3. 计算截至 T-1 日的截面因子。
4. 使用 `train_ic_blend` 计算综合得分。
5. 选择得分最高的 3 只股票。
6. 使用 `score` 方式按得分加权分配资金。
7. 将买入数量调整为 100 股整数倍。
8. 保存每日建议 JSON 和回测明细。

## 7. 资金分配

当前默认分配方式为 `score`：

```text
weight_i = score_i / sum(score)
amount_i = available_capital * weight_i
volume_i = floor(amount_i / price_i / 100) * 100
```

系统仍保留 `equal` 等权分配作为实验参数，但最终默认采用 `score`，因为跨窗口测试显示得分加权更适合当前 3 只集中持仓策略。

## 9. 已淘汰或仅保留为实验的策略

以下方法仍可通过参数运行，但不再作为默认策略：

| 方法 | 当前定位 |
|---|---|
| `xgb` | 保留为模型分数实验项 |
| `ic_blend` | 2026 长窗口可转正，但回撤偏高，跨窗口稳定性不足 |
| `recent_ic_blend` | 近期 IC 因子组合，测试后未优于默认 |
| `ensemble_blend` | 固定融合策略，部分窗口不稳定 |
| `adaptive_blend` | 自适应策略，扩展窗口中出现过度切换，不作为默认 |
| `max_drawdown_stop` | 可选风控参数，默认关闭 |
| `max_stock_price` | 可选高价股过滤，测试中默认不启用 |

## 10. 可解释性总结

当前策略可解释性体现在：

- 因子含义明确，均来自日频量价数据。
- 因子方向来自 IC 验证，不是主观设定。
- 默认策略由多窗口回测筛选得到。
- 回测报告包含收益、回撤、夏普、索泰诺、卡玛、胜率、盈亏比。
- 每日输出可追踪到具体得分、资金分配和持仓数量。

一句话概括：

> AlphaAgent 当前量化层是一个以 Train-IC 因子方向为核心、以分数加权资金分配为执行方式、以多窗口回测为验证标准的可解释 A 股选股策略。
