# 因子构建与量化选股策略说明

> 本文档聚焦 AlphaAgent 量化层的**因子构建逻辑**与**选股决策链路**，目的是增强策略的可解释性——让每一只股票"为什么被选中"都可追溯、可复现、可审计。
>
> 文档内容严格对应实际代码实现（`src/factors/`、`src/models/`），并在末尾如实标注当前局限。

---

## 一、设计哲学

短周期 A 股选股中，**量价信息的信息含量最高、噪声相对可控**，且不依赖财报披露节奏。因此本系统的核心因子全部来自日频前复权 K 线（开高低收、成交量、成交额、换手率），不引入任何盘中或当日数据。

整个量化层遵循三条原则：

1. **可复现**：所有因子都是确定性公式，相同输入必得相同输出，无随机性。
2. **防未来函数**：T 日的决策只使用截至 **T-1 日收盘**的数据。回测中决策截止日严格设为前一交易日（见 `backtest_stock_agent.py` 的 `as_of_date`）。
3. **可解释**：每个因子都有明确的金融直觉，候选池输出附带 `key_factors` 摘要，可逐只回溯驱动因子。

## 二、数据基础

| 项目 | 取值 | 原因 |
|------|------|------|
| 频率 | 日频 | 赛制为日内单向交易 |
| 复权方式 | 前复权 `qfq` | 除权日不复权会导致动量/波动因子严重失真 |
| 主数据源 | baostock | 网络稳定，不依赖东方财富 HTTPS |
| 回看窗口 | ≥ 22 个交易日 | 20 日动量需要 `close[t-1] / close[t-21]`，不足 22 行的标的直接跳过 |

> **成交量单位换算**：baostock / AKShare 的成交量单位为"手"（1 手 = 100 股）。计算 VWAP 时统一换算为股数：`vwap = amount / (volume × 100)`。

## 三、量价因子（13 个）

实现见 [`src/factors/price_volume.py`](src/factors/price_volume.py) 的 `_compute_factors()`。取截至 T-1 日的最新一行因子值。

### 3.1 动量与反转

| 因子 | 公式 | 金融直觉 |
|------|------|---------|
| `mom_1d` | `close[t-1] / close[t-2] − 1` | 最近 1 日涨幅，短期惯性 |
| `mom_5d` | `close[t-1] / close[t-6] − 1` | 周线级别动量 |
| `mom_20d` | `close[t-1] / close[t-21] − 1` | 月线级别动量（同时作为中性化基准） |
| `rev_1d` | `− mom_1d` | 短期反转。A 股短周期常呈现"涨多回调"的反转效应 |

> 动量与反转方向相反，由 XGBoost 在训练中自行学习二者的权重与适用场景，而非人工设定。

### 3.2 波动率

| 因子 | 公式 | 金融直觉 |
|------|------|---------|
| `vol_5d` | `std(日收益率, 5)` | 近期波动幅度 |
| `vol_20d` | `std(日收益率, 20)` | 中期波动幅度（中性化基准） |
| `vol_ratio` | `vol_5d / vol_20d` | **波动率加速度**。>1 表示近期波动放大，常伴随趋势启动或风险释放 |

### 3.3 换手率

| 因子 | 公式 | 金融直觉 |
|------|------|---------|
| `turn_1d` | 当日换手率 | 交投活跃度 |
| `turn_5d_avg` | `mean(换手率, 5)` | 近期平均活跃度（中性化基准） |
| `turn_ratio` | `turn_1d / turn_5d_avg` | **相对换手**。>1 放量，<1 缩量；放量突破往往是资金介入信号 |

### 3.4 量价关系

| 因子 | 公式 | 金融直觉 |
|------|------|---------|
| `price_vol_corr_5` | `corr(close[-5:], volume[-5:])` | 价量配合度。价升量增（正相关）是健康上涨 |
| `price_turn_corr_5` | `corr(close[-5:], turn[-5:])` | 价格与换手的协同 |
| `vwap_dev` | `(close[t-1] − vwap) / vwap` | 收盘价相对成交均价的偏离，反映多空力量对比 |

## 四、AlphaNet 时序特征（10 个）

实现见 [`src/factors/alphanet_features.py`](src/factors/alphanet_features.py)。受 AlphaNet 研究启发，用一组**通用时序算子**从原始序列中提取非线性组合特征，而非人工枚举所有特征形式。

### 4.1 六类时序算子

| 算子 | 含义 |
|------|------|
| `ts_corr(X, Y, d)` | 过去 d 天 X 与 Y 的 Pearson 相关系数 |
| `ts_cov(X, Y, d)` | 过去 d 天协方差 |
| `ts_stddev(X, d)` | 过去 d 天标准差 |
| `ts_zscore(X, d)` | 当前值相对过去 d 天均值的标准化偏离 |
| `ts_return(X, d)` | 过去 d 天涨跌幅 |
| `ts_decaylinear(X, d)` | 线性衰减加权均值（近期权重更高） |

原始输入序列：`open, high, low, close, volume, amount, turn, vwap, return_1d`。

### 4.2 系统采用的 10 个组合特征

| 特征 | 算子组合 | 业务含义 |
|------|---------|---------|
| `corr_close_vol_5` | corr(close, volume, 5) | 5 日价量配合度 |
| `corr_close_turn_5` | corr(close, turn, 5) | 5 日价格-换手配合度 |
| `corr_return_vol_5` | corr(return_1d, volume, 5) | 涨幅与放量的同向性 |
| `std_close_10` | std(close, 10) | 10 日价格波动 |
| `std_vol_10` | std(volume, 10) | 10 日成交量波动 |
| `zscore_close_10` | zscore(close, 10) | 价格偏离近期均值的程度 |
| `zscore_vol_5` | zscore(volume, 5) | 成交量相对水平（放量识别） |
| `ret_close_10` | return(close, 10) | 10 日中期动量 |
| `decay_vol_5` | decaylinear(volume, 5) | 近期放量的加权强度 |
| `decay_close_5` | decaylinear(close, 5) | 近期价格的加权走势 |

> 这些特征与第三章的人工量价因子互补：人工因子提供明确直觉，AlphaNet 特征捕捉量价序列中的非线性结构，为模型提供增量信息。

## 五、五因子 OLS 截面中性化

实现见 [`src/factors/factor_neutralize.py`](src/factors/factor_neutralize.py)。

### 5.1 为什么要中性化

原始因子常隐含**风格暴露**：例如动量因子可能只是变相押注小盘股或某个热门行业。若不剔除，选股结果会系统性偏向特定风格，在风格切换时大幅回撤。中性化的目的是**剥离已知风格的影响，保留因子的纯 alpha**。

### 5.2 方法

对每个交易日的**截面数据**，以下列 5 类变量为自变量做 OLS 回归，取**残差**作为中性化后的因子值：

1. 行业（申万一级 28 行业哑变量，非 28 类归为"其他"）
2. 对数总市值 `ln(market_cap)`
3. 20 日动量 `mom_20d`
4. 20 日波动率 `vol_20d`
5. 20 日平均换手率 `turn_5d_avg`

```
中性化后因子 = 原始因子 − OLS(原始因子 ~ 行业 + ln市值 + 动量 + 波动 + 换手) 的拟合值
```

被用作回归自变量的 `mom_20d / vol_20d / turn_5d_avg` 本身不再被中性化（避免对自己回归）。

> **降级处理**：baostock 路径下行业与市值字段缺失（为 `未知` / `NaN`），此时回归自动跳过对应自变量，仅用可得变量中性化。代码对样本不足或回归失败的因子保持原值，保证流程不中断。

## 六、XGBoost 排序打分

实现见 [`src/models/xgboost_scorer.py`](src/models/xgboost_scorer.py)。

### 6.1 为什么用排序学习而非回归

选股的本质是**截面排序**（"哪些股票明天相对更强"），而非预测绝对收益率。因此采用 `objective='rank:pairwise'`，直接优化排序质量（NDCG），而不是拟合收益率数值——后者会被噪声主导。

### 6.2 标签构建

`build_labels()` 将每日截面的次日收益率分成 10 档（`pd.qcut`），0 = 最差，9 = 最优。模型学习的是"把哪些股票排在前面"。

### 6.3 关键超参数

```python
{
    "objective": "rank:pairwise",   # 排序目标
    "eval_metric": "ndcg",
    "n_estimators": 200,
    "max_depth": 4,                 # 浅树，抑制过拟合
    "learning_rate": 0.05,
    "subsample": 0.8,
    "colsample_bytree": 0.8,
}
```

模型支持初始训练 `train()`、推理 `predict()`（输出归一化到 0~1）、滚动微调 `fine_tune()` 三种模式。

## 七、端到端选股决策链

以 `StockScreener.run()`（[`src/models/stock_screener.py`](src/models/stock_screener.py)）为例，一只股票从全市场到候选池经过：

```
全市场 ~5000 只
   │  ① 拉取股票列表（baostock）
   ▼
过滤不可交易
   │  ② 剔除 ST / *ST / 退市（is_st 关键字）
   │     剔除市值 < 20 亿、PE < 0（字段可得时）
   ▼  约 4970 只
计算因子
   │  ③ 量价因子(13) + AlphaNet 特征(10)
   │     数据不足 22 行的标的在此被跳过
   ▼
五因子中性化
   │  ④ 剥离行业 / 市值 / 动量 / 波动 / 换手 风格
   ▼
XGBoost 打分
   │  ⑤ predict() → 0~1 综合得分（无模型时等权 0.5）
   ▼
排序取 Top-N
   │  ⑥ 按得分降序，输出候选池 + key_factors 摘要
   ▼
候选池（含可解释信息）
```

候选池每一项的数据结构：

```python
{
    "symbol": "600519",
    "symbol_name": "贵州茅台",
    "xgb_score": 0.847,
    "key_factors": {"mom_5d": 0.032, "corr_close_vol_5": 0.71, "zscore_vol_5": 1.4},
    "industry": "食品饮料",
    "market_cap_bn": 2100
}
```

随后进入资金分配层（[`position_sizer.py`](src/execution/position_sizer.py)）：按 `C_mixed` 加权、留 10% 缓冲、单只 ≤ 30%、`volume` 取 100 整数倍，最多持仓 5 只。

## 八、可解释性保障

| 机制 | 说明 |
|------|------|
| `key_factors` 摘要 | 候选池每只股票附带关键因子值，直接回答"为何入选" |
| `reasoning.md` 审计日志 | [`output_formatter.py`](src/execution/output_formatter.py) 为每日决策生成完整链路文档（得分、因子、资金分配） |
| 回测每日建议 JSON | `logs/advice_<tag>/YYYYMMDD.json` 记录每个交易日的操作，可逐日复盘 |
| 防未来函数 | 决策截止日 = T-1，回测中严格执行，杜绝前视偏差 |
| 确定性公式 | 所有因子无随机性，结果可复现 |
| SHAP 因子贡献（规划） | 训练模型后可用 SHAP 量化各因子对单次选股的贡献度 |

## 九、当前局限（如实说明）

为保证文档与实际代码一致，明确标注尚未完成的部分：

1. **XGBoost 需先训练后才有 alpha**。已提供训练脚本 [`train_xgboost.py`](src/models/train_xgboost.py)（特征与选股/回测端完全一致），训练后主流程与回测会自动加载 `data/xgb_scorer.model`。但仓库默认不含训练好的模型文件，未加载时 `_score()` 给所有股票等权 `0.5`，**此时不代表真实量化 alpha**。
2. **资金流因子未接入**。[`capital_flow.py`](src/factors/capital_flow.py) 已实现主力净流入、超大单等因子，但尚未纳入 `StockScreener` 的因子列，当前不参与打分（且依赖 AKShare，历史深度有限）。
3. **智能体层为占位实现**。Layer 3 的 `C_mixed` 当前直接等于 `xgb_score`，多空辩论与 LLM 第三方评分尚未真正接入。

> 因子一致性：训练（`train_xgboost.py`）、回测（`backtest_stock_agent.py`）、选股（`StockScreener.run`）三端共用同一套截面因子计算（`compute_cross_section`：量价 + AlphaNet + 中性化），保证训练特征与推理特征严格对齐。

> 上述局限是迭代路线的下一步重点，详见 [PROJECT_SPEC.md](PROJECT_SPEC.md) 第八章开发路线图。
